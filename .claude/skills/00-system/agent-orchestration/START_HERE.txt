WHAT WILL YOU LEARN?
================================================================================

✓ 6-Phase Pipeline
  Phase 1: Understanding    - ask agents explore codebase
  Phase 2: Research         - research agents learn external knowledge
  Phase 3: Architecture     - architect agent designs solution
  Phase 4: Implementation   - code & frontend agents build it
  Phase 5: Validation       - debug agents test everything
  Phase 6: Documentation    - docs agent writes it up

✓ 8 Agents
  ask       → Explore codebase (read-only)
  research  → Search web (exa, fetch, context7)
  architect → Design architecture, ADR
  code      → Backend Python/FastAPI
  frontend  → React/TypeScript
  debug     → Tests, debugging
  docs      → Documentation
  sre       → Infrastructure, monitoring

✓ Anti-Boomerang Principle
  - Phases = sequential (strict order)
  - Agents within phase = parallel (up to 20 agents)
  - MOI = only orchestrates (you coordinate)
  - Agents never call agents (only MOI invokes)

✓ Performance Gains
  - Audit backend: 20s (parallel) vs 5min (solo) = 15x speedup
  - Features: 2h (pipeline) vs 3h (solo) = 33% faster
  - Bug fixes: 10min (minimal) vs 30min (debug) = 3x faster
  - Tokens: 3,400 (parallel) vs 30,000 (solo) = 89% savings

BASIC INVOCATION TEMPLATE:
================================================================================

Task(agent_name, """
[ULTRA-PRECISE TASK DESCRIPTION]

DEADLINE: [X]s MAX
SCOPE: [Exactly what to do - no more]
PARTIAL OK: [yes/no]
FORMAT: JSON

Output: {...}
""")

Example:
Task(ask, """
Scan backend/services/memory*.py

DEADLINE: 20s
SCOPE: List classes + public methods only
PARTIAL OK: Yes
FORMAT: {files: [...], classes: [...]}

Output: {files: [...], classes: [...]}
""")

================================================================================
ANTI-PATTERNS TO AVOID:
================================================================================

❌ Chain agents (slow):
  Task(ask, "...") → wait
  Task(research, "...") → wait
  Task(architect, "...") → slow!

✅ Parallel independent tasks (fast):
  Task(ask, "backend")
  Task(ask, "frontend")
  Task(ask, "database")
  # All run together = fast!

❌ Vague prompts (agent confused):
  "Analyse le code"

✅ Precise prompts (agent knows exactly):
  "backend/services/memory*.py - list public methods only"

❌ No deadlines (agent takes forever):
  Task(ask, "look at code")

✅ Strict deadlines (agent focuses):
  Task(ask, "... DEADLINE: 20s MAX")

================================================================================
RULES (NON-NEGOTIABLE):
================================================================================

✓ DO:
  - Phases sequential (Understanding → Research → ... → Documentation)
  - Agents parallel (independent tasks run together)
  - MOI orchestrates (only you invoke agents)
  - Ultra-precise prompts (tell agent exactly what to do)
  - Strict deadlines (20s, 30s, 60s - no open-ended)
  - Aggregate results (JSON from agents → your decision)

✗ DON'T:
  - Agent → Agent (only MOI invokes)
  - No deadlines (agent wastes time)
  - Vague prompts (agent confused)
  - Write 1 file per agent (risk conflicts)
  - Circular dependencies (A→B→C→A)
